# Architectural & Tech Stack Overview

## **Frontend Architecture**

### **Next.js & React**
* **Hybrid Rendering Model:** Utilizes the App Router to mix Server-Side Rendering (SSR) for dynamic, SEO critical product pages and Static Site Generation (SSG) for static assets.
* **Optimized Performance:** Minimizes client side JavaScript bundles using React Server Components (RSC) to fetch data directly on the server, ensuring excellent Core Web Vitals.

---

## **Backend & Database Architecture**

### **Go (Golang)**
* **Compiled Efficiency:** Compiles directly to machine code, delivering near native execution speeds and highly predictable performance.
* **High Concurrency:** Uses lightweight Goroutines to handle massive traffic and concurrent requests with minimal memory overhead.

### **Cloudflare D1**
* **Edge-Native SQL:** A serverless relational database built on SQLite that runs natively across Cloudflare’s global network.
* **Ultra-Low Latency:** Positions data physically close to the compute layer, eliminating traditional database connection overhead and complex connection pooling.

---

## **Deployment & Infrastructure**

### **AWS Lambda (ARM Linux)**
* **AWS Graviton Support:** Built specifically for Linux ARM64 (`GOOS=linux GOARCH=arm64`), leveraging ARM-based architecture to deliver up to 34% better price-performance compared to x86.
* **Serverless Scale:** Functions scale automatically in response to incoming traffic, ensuring high availability without the overhead of server management.
* **Sub-Millisecond Cold Starts:** Capitalizes on Go's single-binary compilation and minimal footprint to keep serverless initialization times exceptionally low.
